## What We Learned

- split Cloudflare accounts create unnecessary deployment friction
- the public surface is stronger as an honest object than as a startup-style landing page
- the first wedge is clearer when framed as `scope enforcement + minimal review artifact`
- even a small UI change becomes easier to reason about when the required artifact set is explicit
- the first code-level wedge can stay small if it starts from structured input and bounded output
- stack choice matters early for CLI products, and `single binary` narrows the sane options fast
- pass-only examples are not enough; the wedge becomes more believable once drift is shown explicitly
- external changed-file manifests are a better bridge to real task runners than hand-entered changed paths
- patch-file input is a useful middle step between manifests and direct VCS integration
- stdin support is the cheapest real bridge from repo tooling to the wedge without baking `git` assumptions into the CLI yet
- a non-git workspace blocks true dogfooding of VCS adapters even when the adapter itself is implemented and tested
