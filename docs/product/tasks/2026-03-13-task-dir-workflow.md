# Task Dir Workflow Task

Last updated: 2026-03-13
Owner: Vitaly
Status: completed

## Task

Create the first product-path for minimal task artifacts so Specpunk no longer depends on hand-written `input.json` paths in every command.

## Why This Task

The wedge was already proven across multiple repos and contexts.
The next step is not another proof packet.
The next step is a product path that makes the minimal artifact easier to create and easier to run.

Allowed repo scope:

- `cmd/specpunk/main.go`

- `internal/taskdir/**`
- `docs/product/tasks/2026-03-13-task-dir-workflow.md`

- `internal/check/**`
- `site/**`
- `docs/research/**`

## Outcome

- added `specpunk task init` to scaffold a minimal task directory
- added `specpunk check --task-dir` to run from a task directory instead of a direct `input.json` path
- made `check --task-dir` default its output to `generated-review.md` inside that directory
- added a repo-local demo task directory generated by the new command
- confirmed one real UX rule for shell usage: glob patterns should be quoted when passed to `--allow` and `--block`
