# Artifact Drawer Task

Last updated: 2026-03-12
Owner: Vitaly
Status: completed

## Task

Make the transition from the first review scene to the artifact pack real by adding a minimal artifact drawer flow to the public surface.

## Why This Task

The product message says the artifacts are the object.
Before this change, the page only showed artifacts as a section lower on the page.
This task makes the artifact boundary interactive enough to feel like a review object, not just content below the fold.

## Runtime Scope

Allowed repo scope:

- `site/index.html`
- `site/style.css`

Blocked:

- `docs/research/**`
- `wrangler.toml`
- `site/_headers`
- any framework, build tool, or backend behavior

## Outcome

- the first screen now exposes a real `open artifact drawer` action
- the artifact section now has `open required`, `expand all`, and `collapse all`
- required artifacts are visually grouped as the minimal boundary
- the artifact section receives a visible focus pulse when entered through the drawer flow
