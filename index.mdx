SpecPunk is a local-first runtime for AI-driven engineering work across repositories. It focuses on bounded execution, deterministic gate semantics, proof-bearing decisions, and a simple operator shell.

## Current status

SpecPunk is in an active rebuild phase. These public docs describe the current operator path and keep it separate from future or inactive surfaces.

Today:

- `punk` is the public CLI.
- The active v0 path is `init -> start/go -> plot -> cut -> gate -> proof`.
- `punk-council` is in-tree but inactive.
- `punk-shell`, `punk-skills`, `punk-eval`, and `punk-research` are planned only.

## Principles

- one CLI
- one vocabulary: `plot / cut / gate`
- contract first
- proof before acceptance
- `gate` writes the final decision
- local-first, VCS-aware operation

## Scope of this site

This site is a public, curated layer over the repository. It focuses on what the project is, how to use the current CLI, and what is on the roadmap. Internal research notes are intentionally not part of the public docs.
